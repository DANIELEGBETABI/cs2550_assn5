Concentration Game

Project Checkpoint

Main Logic

Game Board Setup

The game board is created using form information provided by the user. This gathers the size and color of the game board. Currently the board takes this information creats card pairs of 1-n and scatters them across the board.

User Logic

The code to keep track of user turns is in place. This keeps track of how many turns the user has taken, how many pairs they have found, and the time in which it took to find them.

View Logic

The view code is in place to gather information from the board's controller, translate it into a 2d array, and then print that 2d array as cards to the screen. Addclick handlers have been established, and function to reveal the card number when clicked.

Remaining Logic

Turn Logic

The code to execute a turn needs to be established. If a user finds a match, both cards should stay face up and the User's pair variable incremented. If the two cards chosen are not a match, the cards should show the numbers for a few seconds, then revert back to being face down.

End of Game Logic

The logic needs to be written to determine when the end of a game has been met, then alert the user. This can be done by comparing the number of pairs found, compared to the total number of cards/2. When these values are equal, the game has ended.
